Abstract

UI-Vision is a desktop-centric GUI benchmark for visual perception and interaction. It features densely annotated screen UIs from 83+ desktop and open-source platforms, with human-verified annotations for multi-step interactions. The benchmark tests critical skills including UI grounding, layout/spatial relation reasoning, and UI navigation through action prediction.

Types of tasks in the UI-Vision benchmark

UI Data & Benchmark Tasks

The benchmark includes human-created dense UI annotations and focuses on three primary task types:

Action Prediction

Models must predict the correct sequence of actions to accomplish a given task. For example:

  • Adding text "COOL" to an image with the Text tool
  • Following multi-step sequences like MOVE_TO, CLICK, DRAG, TYPING, etc.

Element Grounding

Models must predict the category and bounding box coordinates of UI elements, such as:

  • Identifying navigation controls (Ruler, FreeForm selector)
  • Opening specific dialogs (e.g., color selection)

Layout Grounding

Models must understand spatial relationships and identify regions containing specific UI elements:

  • Navigation controls (back, forward, refresh, home buttons)
  • User interaction elements (search bar, filter options, sorting controls)
