UI-Vision is the first large-scale benchmark for evaluating AI models in real-world desktop GUI environments. It provides dense, high-quality annotations for GUI interaction across 83 applications, supporting tasks like element grounding, layout grounding, and action prediction.

Figure 1: UI-Vision Benchmark Overview, key strengths and tasks

Why UI-Vision?

The Challenge

Current GUI automation research focuses on web-based tasks, leaving desktop environments underexplored. The lack of standardized datasets and visual complexity of software interfaces pose significant challenges for AI models.

What UI-Vision Offers

UI-Vision provides a comprehensive, license-permissive dataset with expert-verified annotations, covering diverse software applications. It allows models to be evaluated on realistic GUI tasks, bridging the gap between research and real-world desktop automation.

Figure 2: Software Distribution - Variety of applications included in UI-Vision

Action Prediction

Our most complex task requires models to predict appropriate actions given a task description, a screenshot, and action history. The task evaluates sequential decision-making abilities in GUI environments.

For example, as shown in the image, a model might be asked to:

  1. Create a circle sketch in CAD software
  2. Navigate through menus to find specific functions
  3. Create a pocket using the sketch as a guide
  4. Set parameters like depth (5.00mm)

Success in this task requires understanding both the current state of the interface and the goal-directed sequence of actions needed to complete complex operations across diverse software applications.

UI-Vision spans 83 software applications across 6 domains, capturing real user interactions through screen recordings and expert annotations. The dataset contains over 8,200 query-label pairs and 450 recorded demonstrations.

Benchmark Tasks

Element Grounding

In this fundamental task, models must identify and localize specific UI elements based on textual queries. We evaluate using Intersection over Union (IoU) with the ground truth bounding boxes.

Example query: "Find the save button"

Element grounding serves as a foundation for AI assistants to interact with GUIs, as accurate element identification is essential for executing user commands.

Layout Grounding

This more challenging task evaluates an AI's ability to group UI components into functional regions. Models must identify complete interface sections like toolbars, menus, or content areas.

Example query: "Highlight the main navigation menu"

Layout grounding tests deeper understanding of interface structure and organization—essential for contextually aware interactions and explaining interface functionality.

Action Prediction

Our most complex task requires models to predict the next appropriate action given a task description, a screenshot, and action history. This evaluates sequential decision-making and task planning abilities.

Example task: "Create a new document and save it as 'report.docx'"

Success in this task requires understanding both the current state of the interface and the goal-directed sequence of actions needed to complete a user's request.

UI-Vision provides a comprehensive evaluation framework for assessing AI systems' capabilities in understanding and interacting with real desktop interfaces. By spanning multiple tasks of increasing complexity, it enables researchers to identify specific strengths and weaknesses in current models.

Key Findings

Performance Analysis

Our extensive evaluation reveals that larger models (50B+ parameters) consistently outperform smaller counterparts across all tasks. Notably, UI-TARS-72B achieved the highest overall performance with a mean score of 25.5% across all categories.

Large GUI Agents

The observed performance gap between GUI-specialized and general-purpose models suggests that domain-specific training significantly enhances GUI understanding capabilities.

Task Difficulty

Among the three main task categories, spatial grounding proved to be the most challenging, with an average performance of only 12.8% across all models. Basic grounding tasks saw the highest success rates, with top-performing models reaching up to 31.4%.

This significant performance drop in spatial tasks highlights the difficulty of understanding relative positioning and spatial relationships in complex UI environments, presenting an important area for future model improvements.

Application Domain Impact

Performance varied significantly across application domains. Browser interfaces yielded the highest success rates (48.2% for top models), while creativity software presented the most challenges (21.9% at best).

This domain variance suggests that interface complexity and specialized terminology significantly impact model performance, with creative software's unique elements proving particularly difficult to interpret.

Despite recent advances, even the best models struggle with complex GUI understanding tasks, achieving only about 25% accuracy overall. This benchmark reveals significant room for improvement in visual-language models' ability to interpret and interact with real-world desktop interfaces.
